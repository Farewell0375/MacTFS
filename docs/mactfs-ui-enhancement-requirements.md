# macTFS 工作台增强需求（对照 Visual Studio 源代码管理）

## 一、背景

前端阶段（FE-001 ~ FE-013）完成后，用户按真实业务试用工作台，对照 Visual Studio
Source Control Explorer 提出了一批体验与功能缺口。本文档收口这批需求，
作为 `task/frontend` 中 FE-014 ~ FE-017 的需求来源。

## 二、用户反馈与根因分析

### 2.1 获取最新会静默覆盖本地修改（最严重）

- 现象：本地改了文件（未签出），执行「获取最新」后本地改动被直接覆盖，无提示、无对比
- 根因：core `getLatest` 固定使用 `GetOptions.GET_ALL | OVERWRITE`，等价于 VS 的
  「强制获取（覆盖本地）」；安全模式应当用 `GetOptions.NONE`，让本地有改动的文件
  产生 writable / 版本冲突并走冲突处理
- VS 行为：Get Latest 默认不覆盖本地可写文件，提示冲突并允许对比后再决定

### 2.2 目录历史下钻的文件列表没有对象操作

- 现象：目录历史 → 点 changeset 看到影响文件，但这些文件行没有右键菜单
- VS 行为：changeset 明细中的文件可以继续查看历史、查看该版本内容、做对比

### 2.3 缺少强制获取与获取指定版本

- 现象：没有「强制获取最新（覆盖本地）」；也没有「获取此历史版本（把目录/文件回到
  某个 changeset）」
- VS 行为：Get Specific Version 支持选择版本 + Overwrite 选项

### 2.4 历史与 Diff 弹窗偏小、对比不好用

- 现象：弹窗小；长文件没有缩略定位；只有单栏统一视图，没有左右分栏对照
- VS 行为：对比窗口左右分栏、有差异概览与上一处/下一处差异导航

### 2.5 右侧挂起更改面板没有手动刷新

- 现象：外部（CLI / 其它工具）产生的挂起更改无法手动拉新
- 同类：左侧树、中间列表也缺少手动刷新入口（FE-012 仅日志面板有刷新）

## 三、我方补充分析（对照 VS 还存在的差距）

| 能力 | VS | 当前 | 本批处理 |
|---|---|---|---|
| 安全获取 / 强制获取 | 区分 | 只有强制 | ✅ FE-014 |
| 获取指定版本 | 支持 | 无 | ✅ FE-014 |
| changeset 明细对象操作 | 支持 | 无 | ✅ FE-015 |
| 目录对比 localOnly「加入版本控制」 | 支持（pend add） | 菜单缺失 | ✅ FE-015 |
| 左右分栏 Diff + 差异导航 | 支持 | 单栏 | ✅ FE-016 |
| 各面板手动刷新 | 支持 | 仅日志 | ✅ FE-017 |
| 重命名 / 移动 / 分支 / Shelve | 支持 | 无 | ❌ 超出第一版范围 |
| 二进制可视化对比 / 三方合并 | 支持 | 无 | ❌ PRD 明确不做 |

## 四、需求定义

### R1 安全获取与强制获取 / 获取指定版本（FE-014）

- core `getLatest` 增加 `force` 参数：
  - `force=false`（默认）：`GetOptions.NONE`，本地改动产生冲突而不是被覆盖
  - `force=true`：维持 `GET_ALL | OVERWRITE`
- core 新增 `getVersion`：按 changeset 获取指定版本（`ChangesetVersionSpec` +
  `GET_ALL | OVERWRITE`，对应 VS Get Specific Version + Overwrite）
- server `/api/files/get-latest` 透传 `force`；新增 `/api/files/get-version`
- 前端：
  - 右键「获取最新」默认安全模式；出现冲突时按既有冲突弹窗流程处理
  - 右键新增「强制获取最新…」，需确认弹窗（明确提示会覆盖本地修改）
  - 历史弹窗（文件与目录）支持「获取此版本…」，需确认弹窗
  - Checkout 前置的 Get Latest 维持安全模式

### R2 历史与对比结果的对象操作完善（FE-015）

- changeset 文件列表行提供右键菜单：
  - 查看此版本内容（按 changeset 读文件内容）
  - 查看该文件历史（切换历史弹窗目标为该文件）
  - 获取此版本…（带确认，复用 R1）
- 目录对比结果 `localOnly` 项菜单提供「加入版本控制」（pend add），
  完成后刷新挂起更改

### R3 Diff 与历史弹窗体验优化（FE-016）

- 历史 / Diff / 目录对比弹窗尺寸提升（接近满屏宽高）
- Diff 支持「统一 / 左右分栏」两种视图切换，左右分栏旧版在左、新版在右
- 增加差异概览条（minimap）：标记差异块位置，点击跳转
- 增加「上一处 / 下一处差异」导航按钮
- 文件查看弹窗同步放大

### R4 工作台手动刷新（FE-017）

- 右侧挂起更改面板头部加刷新按钮
- 中间列表路径栏加刷新按钮（重新加载当前目录）
- 左侧树头部加刷新按钮（重置并重新加载根）

## 五、不在本批范围

- 重命名 / 移动 / 分支 / 合并 / Shelve / Work Item
- 二进制可视化对比、三方合并编辑器
- 多 Workspace 切换 UI
