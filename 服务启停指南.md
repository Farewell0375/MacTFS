# macTFS 前后端启动 / 停止 / 重启指南

本项目运行时由两部分组成：

| 部分 | 是什么 | 端口 |
|---|---|---|
| 后端 | Java 本地 API 服务（`com.mydev.mactfs.server.MacTfsServer`），负责和 TFS 服务器通信 | `127.0.0.1:38765` |
| 前端 | Electron 桌面应用（开发模式下由 vite 提供页面） | 开发页面 `localhost:5173` |

> 关系说明：启动 Electron 桌面应用时，如果发现后端没在运行，它会自动把后端拉起来；退出 Electron 时，会自动回收由它拉起的后端进程。所以日常开发**只需要启动前端**即可。

---

## 一、日常开发：一条命令全部启动

```bash
cd /Users/fenghp/Desktop/DEV/project/mydev/mactfsui
pnpm electron:dev
```

这条命令会同时启动：

1. vite 开发服务器（`localhost:5173`，提供页面，改前端代码自动热更新）
2. Electron 桌面窗口（加载上面的页面，并按需自动拉起后端）

### 停止

在运行该命令的终端按 `Ctrl + C`，或直接退出 Electron 应用窗口。

### 重启

`Ctrl + C` 停掉后重新执行 `pnpm electron:dev` 即可。

## 二、后端单独操作

### 1. 构建（改了 Java 代码后必须先做这一步）

```bash
cd /Users/fenghp/Desktop/DEV/project/mydev/mactfs
../tfsIntegration/gradlew installDist
```

> 注意：后端是从 `mactfs/build/install/mactfs/lib/` 目录加载 jar 包运行的。
> 只执行 `gradlew build` 不会更新这个目录，**必须用 `installDist`**，否则改动不生效。

### 2. 查看后端是否在运行

```bash
lsof -nP -iTCP:38765 -sTCP:LISTEN
```

- 有输出（java 进程）= 正在运行；无输出 = 没有运行。

也可以调用健康检查接口：

```bash
TOKEN=$(cat ~/.mactfs/server-token)
curl -s -H "Authorization: Bearer $TOKEN" http://127.0.0.1:38765/api/health
```

返回 `"success":true` 即正常。

### 3. 手动启动后端

```bash
cd /Users/fenghp/Desktop/DEV/project/mydev/mactfs
nohup /Users/fenghp/Desktop/DEV/project/mydev/zulu8.94.0.17-ca-jdk8.0.492-macosx_x64/Contents/Home/bin/java \
  -cp "build/install/mactfs/lib/*" \
  -Dcom.microsoft.tfs.jni.native.base-directory=build/install/mactfs/lib/native \
  com.mydev.mactfs.server.MacTfsServer > /tmp/mactfs-server.log 2>&1 &
```

- 日志输出在 `/tmp/mactfs-server.log`，启动失败时先看这个文件。
- 固定使用项目内置的 JDK 8（zulu8 目录），不依赖系统 Java。

### 4. 停止后端

```bash
kill $(lsof -t -nP -iTCP:38765 -sTCP:LISTEN)
```

### 5. 重启后端（改完后端代码后的标准流程）

```bash
cd /Users/fenghp/Desktop/DEV/project/mydev/mactfs
../tfsIntegration/gradlew installDist          # 1. 重新构建并更新运行目录
kill $(lsof -t -nP -iTCP:38765 -sTCP:LISTEN)   # 2. 停掉旧服务（没在运行会报错，可忽略）
nohup /Users/fenghp/Desktop/DEV/project/mydev/zulu8.94.0.17-ca-jdk8.0.492-macosx_x64/Contents/Home/bin/java \
  -cp "build/install/mactfs/lib/*" \
  -Dcom.microsoft.tfs.jni.native.base-directory=build/install/mactfs/lib/native \
  com.mydev.mactfs.server.MacTfsServer > /tmp/mactfs-server.log 2>&1 &   # 3. 启动新服务
```

> 重启后端**不需要**重启前端：登录用的 token 保存在 `~/.mactfs/server-token`，重启后保持不变，已打开的桌面应用会继续正常工作。

## 三、前端单独操作

### 仅启动页面（不开 Electron 窗口，浏览器调试用）

```bash
cd /Users/fenghp/Desktop/DEV/project/mydev/mactfsui
pnpm dev
```


等价于 `pnpm build`（打包页面）+ `pnpm electron`（用打包产物启动桌面应用）。

## 四、常见问题

| 现象 | 原因与处理 |
|---|---|
| 改了 Java 代码但行为没变 | 忘了 `installDist`，或没重启后端。按「二、5 重启后端」流程操作 |
| 后端起不来，提示端口占用 | 已有旧进程占着 38765，先执行「二、4 停止后端」再启动 |
| 桌面应用提示找不到服务端构建产物 | 还没执行过 `installDist`，按「二、1 构建」操作 |
| 接口全部返回 401 | token 不一致，确认请求头使用 `~/.mactfs/server-token` 中的最新值 |
| 想看后端报错日志 | 手动启动时看 `/tmp/mactfs-server.log`；由 Electron 自动拉起的后端不输出日志，建议手动启动排查 |
