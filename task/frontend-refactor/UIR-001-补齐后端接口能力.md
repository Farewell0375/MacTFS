## 目标

补齐前端重构所需的后端接口能力，确保文件查看、服务器内容读取、默认 Workspace、目录对比、Get Latest / Checkout 冲突处理可以被 UI 调用。

## 实现范围

- 登录后保存上次使用的 Collection。
- 根据 Collection 自动查找本机已有 Workspace。
- 本机没有 Workspace 时按程序固定规则创建默认 Workspace。
- 返回当前实际使用的 Workspace 名称和基础信息。
- 提供 Mapping 目录内本地文件内容读取能力，并校验路径不能越过 Mapping 目录。
- 提供未映射文件的服务器 latest 内容读取能力。
- 文件内容接口返回文件大小、是否二进制、是否可文本渲染、文件编码和内容。
- 文件超过 5MB 时返回大小信息，前端不直接渲染内容。
- Diff 接口支持本地文件与服务器 latest、两个历史 changeset。
- 目录对比接口返回差异状态、可执行动作、pending edit 子项。
- Get Latest 支持递归、跳过 pending edit 子文件并返回跳过列表。
- Checkout 自动先 Get Latest，再签出。
- Get Latest / Checkout 遇到冲突时返回统一冲突明细。
- 冲突明细包含服务端路径、本地路径、文件大小、是否二进制、是否可自动合并。
- 支持应用冲突选择：使用服务器版本、保留本地版本、自动合并结果。
- Pending Changes 返回 pending add / edit / delete 类型，支持菜单判断。

## 不在范围

- 不实现前端页面。
- 不实现 Monaco 或 CodeMirror。
- 不做手动冲突块编辑。
- 不改变现有 TFS 协议封装以外的业务规则。

## 验收标准

- 登录连接后能返回固定 Collection 和当前 Workspace。
- 没有 Workspace 时能创建默认 Workspace。
- 已映射文件能读取本地内容，未映射文件能读取服务器内容。
- 非 Mapping 目录路径读取被拒绝。
- 大于 5MB 文件不返回正文渲染内容。
- pending add 文件不会提供服务器比较能力。
- Get Latest / Checkout 冲突能返回前端弹窗所需明细。
- 应用“使用服务器版本”后无 pending change。
- 应用“保留本地版本”后进入 pending edit。
- 自动合并结果确认应用后进入 pending edit。
