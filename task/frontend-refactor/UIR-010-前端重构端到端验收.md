## 目标

对前端重构后的核心流程做端到端验收，确认工作台布局、右键菜单、弹窗、文件查看、Diff、冲突处理、Checkin 都符合设计方案。

## 实现范围

- 验证登录页 Collection 默认选中和固定上下文。
- 验证 Workspace 自动使用或创建。
- 验证工作台左右和底部收起。
- 验证左侧树和中间文件列表同步导航。
- 验证左侧树、文件列表、Pending Changes 右键菜单。
- 验证 Mapping 弹窗。
- 验证 History 弹窗。
- 验证目录对比弹窗和状态筛选。
- 验证文件查看弹窗代码高亮、搜索、行号。
- 验证 Diff 弹窗左右分栏、搜索、行号、差异高亮。
- 验证 Get Latest 递归、pending edit 跳过、冲突弹窗。
- 验证 Checkout 自动 Get Latest 和冲突复用。
- 验证 Pending Changes 右键撤销、查看、比较、Included/Excluded。
- 验证 Checkin 后刷新当前目录和 Pending Changes。

## 不在范围

- 不做 Work Item。
- 不做 Check-in Policy UI。
- 不做手动冲突块编辑。

## 验收标准

- 所有 UIR 阶段任务通过 `pnpm typecheck`。
- Electron 环境下可以完成登录、浏览、Mapping、目录对比、文件查看、Diff、Get Latest、Checkout、Pending Changes、Checkin 基本流程。
- 未映射文件查看服务器内容。
- 已映射文件查看本地内容。
- 大文件不直接渲染。
- 本地非 Mapping 路径不能查看。
- 冲突处理三种选择结果符合文档。
