## 目标

修复「获取最新静默覆盖本地修改」问题：默认获取走安全模式让本地改动进入冲突流程，
强制覆盖与获取指定版本成为带确认的显式动作，对齐 Visual Studio 行为。

## 实现范围

- core `getLatest` 增加 `force` 参数（默认安全 `GetOptions.NONE`，强制 `GET_ALL|OVERWRITE`）
- core 新增 `getVersion(serverPath, changeset, recursive)`（指定版本 + 覆盖）
- server `/api/files/get-latest` 透传 `force`；新增 `POST /api/files/get-version`
- 前端 API 层补充 `force` 与 `getVersion`
- 右键「获取最新」改为安全模式；冲突走既有冲突弹窗
- 右键新增「强制获取最新…」+ 确认弹窗（明示覆盖本地修改）
- 历史弹窗行支持「获取此版本…」+ 确认弹窗
- Checkout 前置 Get Latest 维持安全模式

## 不在范围

- 不做三方合并
- 不做按日期 / 标签的版本规格（仅 changeset）
- 不改冲突弹窗本身

### 实际修改文件

- `mactfs/src/main/java/com/mydev/mactfs/core/MacTfsCoreService.java`：`getLatest` 增加 `force` 重载（默认 `GetOptions.NONE` 安全模式，强制为 `GET_ALL|OVERWRITE`）；新增 `getVersion`（`ChangesetVersionSpec` + 覆盖获取）
- `mactfs/src/main/java/com/mydev/mactfs/server/MacTfsServer.java`：`/api/files/get-latest` 透传 `force`；新增 `POST /api/files/get-version`
- `mactfsui/app/lib/api/endpoints.ts`：`getLatest` 支持 `force`；新增 `getVersion`
- `mactfsui/app/lib/tfs/actions.ts`：新增 `forceGetLatest` 动作（目录 / 文件菜单均提供，危险态）；安全模式下文件「获取最新」不再因挂起状态置灰
- `mactfsui/app/hooks/use-file-actions.ts`：`runGetLatest` 支持 force；新增 `runGetVersion`、`handleForceGetConfirmed`；`forceGetLatest` 动作先弹确认
- `mactfsui/app/components/app/confirm-dialog.tsx`（新增）：通用危险操作确认弹窗
- `mactfsui/app/components/app/workspace-dialogs.tsx`：渲染强制获取确认弹窗；向历史弹窗注入 `onGetVersion`
- `mactfsui/app/components/explorer/history-dialog.tsx`：每条历史记录提供「获取此版本」按钮 + 确认弹窗

### 实际实现内容

- 修复用户反馈最严重的问题：「获取最新」不再静默覆盖本地修改。默认安全模式下本地有改动的文件会产生冲突，自动进入既有冲突弹窗（可对比后选择取舍）
- 「强制获取最新…」为显式危险动作：菜单红色样式 + 确认弹窗明示「本地未签入的修改会被覆盖且无法恢复」
- 历史弹窗（文件 / 目录）每条记录可「获取此版本」（对应 VS Get Specific Version + Overwrite），同样有确认弹窗；完成后刷新目录列表与挂起更改
- Checkout 前置 Get Latest 维持安全模式
