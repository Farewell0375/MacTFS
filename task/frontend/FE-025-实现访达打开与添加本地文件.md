## 目标

补齐两个日常操作缺口：右键对象「在访达中打开」（文件定位选中、目录直接进入），
以及目录右键 / 中间区域空白处右键「添加本地文件…」（把本地新增文件加入 TFS 版本控制）。

## 实现范围

- Electron main 新增 IPC：`reveal-path`（`shell.showItemInFolder` / `shell.openPath`）、
  `select-files`（系统多选文件选择器，支持默认目录）
- preload / bridge / 类型契约同步暴露 `revealPath`、`selectFiles`
- 动作模型新增 `revealInFinder`、`addLocalFiles`：
  - 文件菜单「在访达中显示」；目录菜单「在访达中打开」（未映射置灰）
  - 目录菜单「添加本地文件…」（未映射置灰）
- 中间文件列表空白区域右键 = 当前目录的完整右键菜单（行内右键阻止冒泡避免双菜单）
- 添加流程：文件选择器（默认定位目录本地路径）→ 过滤越界文件（提示跳过）→
  `POST /api/files/add` → 刷新挂起更改与目录列表
- 非 Electron 环境：两个动作给出「仅桌面应用支持」提示

## 不在范围

- 不做拖拽添加文件
- 不做新建文件 / 文件夹
- 不改既有目录对比 localOnly「加入版本控制」入口

## 完成记录

### 实际修改文件

- `mactfsui/electron/main.cjs`：新增 IPC `mactfs:reveal-path`（目录 `shell.openPath` 直接进入，文件 `shell.showItemInFolder` 定位选中，路径不存在返回 false）、`mactfs:select-files`（系统多选文件选择器，支持默认目录）
- `mactfsui/electron/preload.cjs`：暴露 `revealPath`、`selectFiles`
- `mactfsui/app/lib/electron/types.ts` / `bridge.ts` / `index.ts`：桥接契约与封装（非 Electron 环境优雅降级）
- `mactfsui/app/lib/tfs/actions.ts`：`FileActionId` 新增 `revealInFinder`、`addLocalFiles`；文件菜单「在访达中显示」、目录菜单「在访达中打开」+「添加本地文件…」（未映射置灰）
- `mactfsui/app/hooks/use-file-actions.ts`：两个动作的执行逻辑（访达打开失败提示先获取最新；添加流程过滤目录外文件并提示跳过，成功后刷新挂起更改与目录列表）
- `mactfsui/app/components/explorer/folder-items-panel.tsx`：中间区域空白处右键 = 当前目录完整菜单（行内右键 stopPropagation 防止双菜单）

### 实际实现内容

- 「在访达中打开 / 显示」：目录直接进入访达对应目录；文件在访达中定位并选中；本地路径不存在（未下载）时提示先获取最新；非 Electron 提示仅桌面应用支持
- 「添加本地文件…」：文件选择器默认定位该目录本地路径，多选；只接受位于该目录本地路径内的文件（越界跳过并提示）；调用既有 `POST /api/files/add` 产生 pendingAdd，挂起面板可见、签入后生效
- 中间列表空白处右键提供当前目录的完整右键菜单，与目录树右键一致
