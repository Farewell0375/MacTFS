## 目标

实现底部操作台和统一刷新反馈，让长耗时操作和失败结果在 UI 中可追踪。

## 实现范围

- 底部日志面板
- 每次主要操作的日志展示
- 开始时间、结束时间、耗时、成功 / 失败、错误摘要
- 长操作期间的显式 loading 文案
- 关键操作后的最小刷新策略落地

## 不在范围

- 不实现日志导出
- 不做实时流式日志
- 不做异步 Job 与取消按钮
- 不做全局通知中心

- 主要操作都有对应日志记录
- Get Latest、Compare、Checkin 等长操作有明确执行中提示
- 失败时可以看到可读错误摘要

## 完成记录

### 实际修改文件

- `mactfsui/app/components/logs/console-panel.tsx`（重写）：底部操作日志面板
- `mactfsui/app/hooks/use-file-actions.ts`：动作编排依赖新增 `refreshLogs`，在 unmap / getLatest / checkout / delete / undo / checkin / 冲突解决后触发日志刷新
- `mactfsui/app/components/app/workspace-shell.tsx`：新增 `logsRefreshToken`，向 Console 透传忙碌状态与执行中文案

### 实际实现内容

- 日志面板：拉取 `/api/logs`，按开始时间倒序展示——成功 / 失败图标、操作名（等宽）、摘要、失败错误摘要（红色）、开始时间、耗时（ms / s 自适应）；手动刷新按钮；空态与加载态
- 自动刷新：每次主要操作（获取最新、签出、挂起删除、撤销、签入、取消映射、冲突应用）完成后日志自动重载
- 执行中提示：长操作期间 Console 头部展示 spinner + 当前执行文案（如「正在获取最新…」），顶部通知条同步显示
- 最小刷新策略已全部落地（连接→config/collections、确认 Collection→workspace/树根、Mapping 增删→mappings+当前目录、Get Latest→当前目录+日志、Checkout/Delete/Undo→pending、Checkin→pending+当前目录），本任务补齐日志维度
