### 实际实现内容

- 服务端新增路由：
  - `GET /api/workspace/context`：返回固定上下文（serverUri / collection / workspace / connected / mappings），由会话配置与 `listMappings` 组合，不变更 TFS 状态。
  - `POST /api/mappings/check-target`：按“父目录 + 服务端路径最后一段”计算最终目标路径并返回 `exists` / `allowed`，纯本地文件系统校验。
  - `GET /api/files/content`：传 `localPath` 读取本地映射文件，否则按 `serverPath` + 可选 `changeset` 读取服务器内容；统一套用二进制和大文件约束。
  - `GET /api/conflicts`：查询当前 Workspace 冲突明细，支持 `serverPath` 范围与 `recursive`。
  - `POST /api/conflicts/apply`：对单个冲突应用 `takeServer`（采用服务器版本）或 `keepLocal`（保留本地版本）。
- 核心层新增 `getLocalFileContent`、`listConflicts`、`applyConflict`，并把 `getFileContent` 改为统一走二进制嗅探（前 8000 字节空字节）+ 2MB 大小阈值的内容构建器；`TfsFileContent` 增加 `size` 与 `tooLarge` 字段；新增 `TfsConflictInfo`、`TfsConflictResolution` 数据类。
- 大文件 / 二进制：`content` 返回空串，`binary` / `tooLarge` 标记为真，前端据此提示。
- 非 Mapping 路径：服务器内容接口仍可读取 latest；本地内容接口在文件不存在时返回明确错误，由前端展示 `errorMessage`。
- 前端类型契约与端点封装一一对应服务端 `data` 结构，组件后续不再散落临时类型与裸 `fetch`。
