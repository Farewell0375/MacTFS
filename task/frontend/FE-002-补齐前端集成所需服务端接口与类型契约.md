## 目标

补齐工作台改版所需的服务端接口和前端类型契约，避免后续页面实现时再拆返。

## 实现范围

- 增加前端固定 Collection / Workspace 上下文所需接口
- 增加文件内容读取接口，覆盖本地映射文件和服务器 latest 文件
- 增加冲突明细与冲突选择应用接口
- 明确大文件、二进制文件、非 Mapping 路径的返回约束
- 在 `app/lib/api/types.ts` 中补齐前端统一类型
- 在 `app/lib/api/endpoints.ts` 中补齐接口封装

## 不在范围

- 不实现具体业务页面
- 不实现 Monaco / CodeMirror
- 不实现手动冲突块编辑
- 不调整无关 Server API 路由

- 前端有稳定的 Workspace 上下文接口可调用
- 前端有稳定的文件内容接口可调用
- 前端有稳定的冲突查询与应用接口可调用
- 大文件、二进制文件、越界路径的返回规则明确
- 前端不再在组件里散落临时接口类型
