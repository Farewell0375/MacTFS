## 目标

为服务端目录 / 文件提供分支能力：从源路径分叉出目标路径（pendBranch 产生挂起更改，
签入后服务器生效），对齐 VS Source Control Explorer 的 Branch。

## 实现范围

- core 新增 `branch(sourceServerPath, targetServerPath, changeset?)`：
  `Workspace.pendBranch`，默认基于 latest，可指定 changeset
- server 新增 `POST /api/files/branch`
- 前端 API 层补充 `branch`
- 右键菜单新增「分支…」（树 / 列表来源）：
  - 弹窗：源路径只读展示；目标路径默认 `源路径-branch` 可编辑；
    可选「基于指定 changeset」（输入或从历史点选，复用 FE-019 版本选择器）
  - 校验：目标路径不能与源相同、不能已存在；目标父目录必须已映射（pendBranch 需本地落盘），
    未满足时给出明确提示
- 分支产生 branch 挂起更改，挂起面板可见、可 undo、可签入（复用既有流程）
- 完成后刷新目录树与挂起更改

## 不在范围

- 不做分支可视化（Branch Hierarchy 图）
- 不做「转换为分支对象」（Convert to Branch）
- 不做合并（FE-024）
