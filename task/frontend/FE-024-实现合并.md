## 目标

提供分支间合并能力：查看待合并变更集候选，执行合并产生挂起更改，
冲突进入既有冲突弹窗，审查后签入，对齐 VS 的 Merge 向导。

## 实现范围

- core 新增：
  - `mergeCandidates(sourceServerPath, targetServerPath)`：`Workspace.getMergeCandidates`
  - `merge(sourceServerPath, targetServerPath, changeset?)`：`Workspace.merge`，
    默认合并全部候选，可指定单个 changeset
- server 新增 `GET /api/files/merge-candidates`、`POST /api/files/merge`
- 前端 API 层补充 `mergeCandidates`、`merge`
- 右键菜单新增「合并…」（树 / 列表来源，作为合并源）：
  - 向导弹窗：源路径只读；目标路径输入（必须已映射，校验给出原因）
  - 加载并展示候选 changeset 列表（编号、提交人、时间、备注）；空列表提示无可合并内容
  - 支持「全部合并」或选择单个 changeset 合并
  - 确认弹窗说明：只产生挂起更改、冲突需要逐个处理、签入前可撤销
- 合并冲突进入既有冲突弹窗流程（listConflicts / applyConflict）
- 完成后刷新挂起更改，引导用户审查并签入

## 不在范围

- 不做三方合并编辑器（内容冲突仍按「取本地 / 取服务器」处理）
- 不做 baseless merge（无分支关系的强制合并）
- 不做合并历史可视化

### 实际实现内容

- 右键「合并…」打开向导：目标路径必须已映射（校验给出原因）；「查询候选」展示待合并 changeset（编号 / 类型 / 作者 / 时间 / 注释），空列表提示已同步
- 支持「全部合并」与「合并所选」（点选候选行选中 / 取消）；执行后顶部通知展示统计并提示去挂起面板审查后签入
- 合并冲突自动进入既有冲突弹窗（listConflicts / applyConflict，按「取本地 / 取服务器」处理）
