## 目标

为选中的文件 / 文件夹提供「高级」能力：属性弹窗（服务器路径、本地路径、映射与挂起状态、
服务器最新版本信息），以及右键直达的「获取特定版本…」（按 changeset 把对象本地内容
整体切到该变更集时刻），对齐 VS 的 Properties 与 Get Specific Version。

## 实现范围

- server 新增 `GET /api/items/info?serverPath=`：返回对象类型、最新 changeset、
  签入时间、签入人，文件补充大小与编码（core 用 `VersionControlClient.getItem`）
- 前端 API 层补充 `itemInfo`
- 右键菜单（文件与目录、树 / 列表 / 挂起 / 对比来源均可见）新增：
  - 「属性…」：弹窗展示服务器路径、本地路径（未映射显示「未映射」）、对象类型、
    映射状态、挂起状态、服务器最新版本信息；路径支持一键复制
  - 「获取特定版本…」：弹窗内 changeset 输入框 + 「从历史选择…」内嵌历史列表点选回填；
    明确提示会覆盖本地内容（目录递归）；确认后调用既有 `/api/files/get-version`
- 完成后刷新目录列表与挂起更改
- 未映射对象：「获取特定版本…」置灰并给出原因

## 不在范围

- 不做按日期 / 标签的版本规格（仅 changeset）
- 不做回滚（产生挂起更改的反做走 FE-021）
- 不改历史弹窗既有「获取此版本」入口
