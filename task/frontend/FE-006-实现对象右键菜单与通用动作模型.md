## 目标

把文件、目录和 Pending Changes 的对象操作统一收口到右键菜单和共享动作模型，避免顶部和列表底部继续堆操作按钮。

## 实现范围

- 左侧目录树节点右键菜单
- 中间文件列表文件 / 文件夹右键菜单
- Pending Changes 项右键菜单
- 目录对比结果项右键菜单约定
- 提炼通用 `FileTarget` / `FileAction` 或等价动作模型
- 按 mapped、folder、pending、localPath 等状态控制菜单可用性
- 左侧树与中间文件列表目录动作保持同一套展示和置灰规则
- 未映射目录保留本地工作区相关动作，但置灰不可点击
- 已映射目录不再展示 `映射到本地`
- 已映射目录支持 `取消映射`

## 不在范围

- 不实现具体弹窗内容
- 不实现 Checkin 弹窗
- 不实现键盘快捷键
- 不实现手动冲突块编辑

## 验收标准

- 左侧树、文件列表、Pending Changes 都可打开右键菜单
- 顶部不再承担文件对象操作
- pending add 文件不显示服务器比较入口
- pending edit 文件不能直接执行危险的 Get Latest 覆盖
- Checkin 仍只在右侧 Changes 面板提交
- 左侧树和中间文件列表同一路径的目录动作一致
- 未映射目录的本地工作区相关动作为置灰不可点击
- 取消映射后当前浏览位置不跳转，只更新动作可用性
