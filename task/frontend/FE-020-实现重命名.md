## 目标

为已映射的文件 / 文件夹提供重命名能力：产生 rename 挂起更改，可撤销，签入后服务器生效，
对齐 VS Source Control Explorer 的 Rename。

## 实现范围

- core 新增 `rename(serverPath, newName)`：`Workspace.pendRename`，对象须已映射；
  返回结果复用 `TfsFileOperationResult`
- server 新增 `POST /api/files/rename`（参数 `serverPath`、`newName`）
- 前端 API 层补充 `rename`
- 右键菜单新增「重命名…」（树 / 列表来源，文件与目录均可）：
  - 弹窗输入新名称，校验：非空、不含 `/ \ : * ? " < > |`、与原名不同
  - 已有挂起 rename 或未映射时置灰并给出原因
- 挂起更改面板展示 rename 状态（`pendingRename`），显示「旧名 → 新名」
- rename 挂起更改支持 undo 撤销、checkin 签入（复用既有流程）
- 完成后刷新目录树、目录列表与挂起更改

## 不在范围

- 不做跨目录移动（Move）
- 不做批量重命名
- 不处理大小写仅变化的特殊冲突场景（记录遗留问题即可）

## 验收标准

- 已映射文件 / 目录右键「重命名…」可输入新名称并提交
- 重命名后产生 rename 挂起更改，挂起面板显示「旧名 → 新名」
- undo 可撤销重命名（本地名称还原）；checkin 后服务器生效
- 非法名称、重复名称、未映射对象有明确校验与置灰提示
- 服务端构建通过、前端 typecheck 通过
